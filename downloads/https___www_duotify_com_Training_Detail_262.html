教育訓練.NET / C# 開發實戰：掌握相依性注入的觀念與開發技巧
1 天
後端.NET技術類 .NET / C# 開發實戰：掌握相依性注入的觀念與開發技巧本課程將透過一個又一個的實務案例，帶大家了解「相依性注入」的應用情境， 過程中也會介紹多套好用 DI 容器與開發技巧，幫讓大家更有效率的使用 DI 容器。

此課程適合給...

  • Web 開發人員 / 軟體工程師 / .NET 開發人員
  • C# 開發人員 (本課程將以 C# 為主要開發語言)
  • 軟體架構師 / 系統分析師 / 系統設計師

上完此課程你將會...

  • 認識 SOLID 物件導向程式設計原則
  • 了解相依性注入技術所帶來的實質效益
  • 了解多套 DI Container 的容器用法與差異
  • 學會相依性注入的開發方式與應用技巧

相依性注入 (DI) 是一個知名的物件導向設計模式，在近幾年也逐漸受到重視， 許多知名的開發框架中都有 DI 的影子。但每次提到物件導向的設計模式 (Design Pattern)， 都會讓許多人望之卻步，就因為有許多設計模式的抽象概念艱澀難懂，即便程式都寫得出來，也不知道真正背後的用意為何，更不知道為什麼要這樣用！

這堂課經過數個月的規劃設計，大幅降低學習門檻，讓大家能深入理解 DI 可以為我們手邊的專案帶來什麼樣的改善。 好的工具，可以幫助你快速達成任務；但好的觀念，才能幫你設計出更好的軟體架構，提高系統的可維護性， 讓開發人員在維護系統的過程更加輕鬆。

本課程將透過一個又一個的實務案例，帶大家了解「相依性注入」的應用情境， 過程中也會介紹多套好用 DI 容器與開發技巧，幫讓大家更有效率的使用 DI 容器。

學員回饋
學員B

因非本科系出身，物件觀念都為自學，在此課程中，重新認識物件導向設計，講師點出了許多最基礎卻又最容易遺忘的概念， 收穫良多。學習與使用DI非一蹴而就，需有多年的程式經驗累績與相關知識的培養， 理解與使用設計模式，且在開發的過程中將 SOLID 原則融會貫通，最後才是使用 DI 套件注入的實體物件， 將鬆散耦合的程式帶入專案之中。期待自己可將 coding 能力提升至此境界。

學員C

第一次上保哥的課，覺得表達清晰流暢，課程中提供的許多舉例都很好吸收。當學員提問時， 會把問題拉高層次引導學員思考問題的本質，除了程式外，更注重的是引導學員如何進行抽象思考， 這個課程可以獲得許多程式思維的釐清，這是我覺得最棒的地方。

學員D

原本對 DI 和一些基本的 DesignPattern 就有點基礎概念，框架部分之前有看過一些文件， 不過因為沒有實際用過，所以也算是蠻陌生的，也讓我懷疑我真的懂 DI 嗎， 上了這次課程真的收穫超多，此次課程讓我覺得茅塞顿開的點就在於親手實作 PureDI ， 由最簡單的概念開始講起搭配實作真的讓我很快就理解框架的核心概念，不過框架的部分版本很多， 可能就是有需要用到再去看文件。
