版本控管技術類 Git 版本控管實戰：新手入門篇以 30天精通Git版本控管 獲得【iT邦幫忙鐵人賽年度大獎】、【開發技術組年度鐵人】兩項殊榮的講師：保哥， 將用生動的教學方式與實際案例，推廣 Git 的精隨及奧妙，大家不要錯過喔。

此課程適合給...

  • Web 開發人員 / 網頁設計師 / 網頁前端工程師 / 軟體工程師 / 專案人員
  • 適合 Windows, Linux, Mac 各種平台的使用者

上完此課程你將會...

  • 了解 Git 核心架構與重要版控觀念
  • 了解 Git 各式基礎指令操作
  • 了解 Git 視覺化工具操作：TortoiseGit、SourceTree、Visual Studio Tools for Git
  • 了解 GitHub 版控機制與協同作業功能
  • 了解如何在公司內部導入 Git 版本控管機制

您曾經遇到，好不容易完成的程式與文件資料在上傳後，卻被告知是『衝突版本』嗎? 或者在團隊工作中，總是碰到同時間 Update ，但又不知道哪一個版本才是最新的窘境嗎？

邀請您一同要來玩玩 Git 這套分散式版本控管系統，它能讓您輕鬆控管程式，不用在花多餘的時間來排除衝突檔案， 隨心所欲的在自己開發環境上進行編輯，最後在合併於主程式中。

以 30天精通Git版本控管 獲得【iT邦幫忙鐵人賽年度大獎】、【開發技術組年度鐵人】兩項殊榮的講師：保哥， 將用生動的教學方式與實際案例，推廣 Git 的精隨及奧妙，大家不要錯過喔。

學員回饋
學員C

之前就有自己看過 git 的教學，不過看到後來，常常遇到許多疑惑的地方， 而保哥的課程，從觀念來引導，搭配一些實作，能清楚其中的脈絡。

學員D

這次 git 課程讓我對版本控制有一個較爲基礎的認識和觀念上的導正， 所以這堂課並不是只針對 git 作介紹而已！加上一些實務上的案例作討論， 以目前各版控工具做法來互相比較其優缺點和建議做法和精神意義。對未來於公司內推動和教育訓練有很大幫助！

學員E

了解了版本控管跟 Source Code 備份是不一樣的，Workspace 跟 Repository 之間的差異， 沒有 commit ，一切都不用說了，切 branch 的理解，還有他的好朋友 checkout ，當然 merge 也是重中之重， 一切的成果就交給他來整合，實務上，衝突難免，即使沒有也要檢視，工具只是簡化協同合作的流程跟提醒使用者， 主要還是在於人的使用上，及人與人之間的溝通。

學員A

在上課之前自己就已經有自學 git，看了幾本書之後，也將 git 利用在工作的版控， 但仍沒辦法產生太大信心，今日最大的收獲是老師將 git 的基本重要觀念: branch/ checkout / reset / push / pul / merge 等，都一一詳盡的說明， 加上平行宇宙/控制時間的概念後，git 對我來說已不再抽象，也更有信心使用 git 這個好工具。

學員B

不了解 git 時，覺得它真的是位高高在上的神，上過課之後覺得自己即將變成神！ 很感謝保哥把 git 的觀念講解得如此易懂，以前在網路上搜索到的文章都不會提到這些觀念， 有提到的也不太能理解，上保哥的課真的學到很多！
