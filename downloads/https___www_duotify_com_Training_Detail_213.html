1 天
版本控管技術類 Git 版本控管實戰：新手進階篇本課程不教基礎 Git 操作，只教更深入的技術原理，過程中也會有許多實務練習主題， 透過大量的練習與即時回饋，幫助大家輕鬆掌握 Git 版本控管。

此課程適合給...

  • Web 開發人員 / 網頁設計師 / 網頁前端工程師 / 軟體工程師 / 專案人員
  • 適合 Windows, Linux, Mac 各種平台的使用者
  • 本課程不適合沒用過 Git 版本控管的使用者

上完此課程你將會...

  • 了解 Git 核心架構與重要版控觀念
  • 了解 Git 分支合併技巧與技術原理
  • 了解 GitHub 版控機制與協同作業
  • 了解如何在公司內部導入 Git 版本控管機制

Git 是一套分散式版本控管系統(DVCS)，有別於傳統集中式版本控管機制， Git 採用了更加先進的邏輯與工具，有效改善不同版本控管情境可能遭遇的問題， 因此正以極快的速度席捲全世界，有越來越多的開發人員漸漸改用 Git 當成主要的版控機制。

以我多年的 Git 版控經驗，我深深了解到，唯有了解 Git 的內部結構與運作原理， 才能真正有自信的掌握 Git 版控，所以你在心中自然產生的不確定性與害怕合併與衝突， 都是不必要的情緒負擔。本次我們特別推出【新手進階篇】全新課程，不教基礎 Git 操作 只教更深入的技術原理，過程中也會有許多實務練習主題，透過大量的練習與即時回饋， 幫助大家輕鬆掌握 Git 版本控管。

學員C

第一次好好地面對 git 這個分散式版控，在上課以前通常都是用指令跑那幾個常用的功能(add, commit, push)， 頂多頂多會有 merge 吧，但上課的過程之中，透過所謂的 Workspace => Index => Repository 的關係， 讓我們能夠透過觀念理解指令真正在 .git 資料夾中所做的異動， 然後實際演練了 Rebase (之前一直很想釐清的一個指令)，目前再有看不懂的指令，我想把 help 叫出來， 那些專有名詞也應該可以讓我看得懂了，感恩保哥讚嘆保哥！

學員D

短短一天就能釐清 git 核心觀念與原理，也只有保哥的課能辦到！

學員E

體驗 git 設計優雅之美 用一個規則，滿足所有需求。上午 git 的基礎，奠定下午的進階用法。 讓我對 git 相關操作有更深入的了解，更了解Commit該如何解讀。同時也導正過去很多錯誤或是似懂非懂的觀念。

學員A

第一次上保哥的課程，真的是既期待又怕受傷害，上課前對 git 很陌生， 也不知道如何活用，上完課後，聽了很多觀念讓我覺得更有信心去征服 git 版本控制， 也知道為什麼要選版控要選 git ，因為 git 的分散式管理，讓版控變得更有彈性, 也不怕誤刪檔案救不回來。
